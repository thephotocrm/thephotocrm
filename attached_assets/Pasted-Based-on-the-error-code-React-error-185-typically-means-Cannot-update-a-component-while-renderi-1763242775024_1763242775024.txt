Based on the error code, React error #185 typically means **"Cannot update a component while rendering a different component."** This is a common issue when you're causing a state update or navigation during the render phase of a component.

Looking at your App.tsx, I can see a few potential issues:

## Most Likely Culprits:

1. **The `<Redirect>` in ProtectedRoutes** - This redirect happens during render and might be causing the issue if the client portal login screen is somehow triggering the ProtectedRoutes component.

2. **Route matching conflicts** - Your route structure has overlapping paths. Notice that you have:
   - `/client-portal` as a public route
   - Then later, all the protected routes that could match

3. **The `useEffect` that opens the onboarding modal** - While less likely, this could be triggering during render in some edge cases.

## Recommended Fixes:

### 2. Check if ClientPortal Component is Causing Navigation During Render

The client portal login screen might be triggering a redirect or navigation during its render. Can you share the ClientPortal component code? The issue is likely there if it's trying to redirect users during render instead of in a useEffect.

The key insight is that React error #185 means something is calling a state update or navigation function **during the render phase** rather than in an event handler or useEffect. This is usually caused by calling navigate/redirect directly in the component body rather than in a useEffect or event handler.
